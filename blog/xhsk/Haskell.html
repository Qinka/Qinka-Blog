Haskell

Haskell is an advanced purely-functional programming language. It began life in the late 1980s as an intened standard language for lazy functional programming. Since it was created, Haskell has gone thorugh various changes, and the current standard of Haskell is Haskell 2010. Haskell was Haskell Curry's first name, and because he, Haskell Curry had already given his name to a process called 'currying' and Haskell is called Haskell.[1]

Haskell is statically typed

Each expression in Haskell will be tagged with a type which is determined at compile time. And the compiler will check the type, and if the types composed together by function application do not match up, the codes will be rejected by compiler.

Haskell has type inference

The types of the functions and expressions will be inferred by unifying every type bidirectionally.

Haskell is lazy

We called it lazy that: When it needed, it is needed. Another meaning is "called by need".

Haskell is purely functional

Each function in Haskell is a function in the mathematical sence. Haskell split the side-effection IO and pure function with IO Monad, so you can prove these pure functions with math.

Haskell is concurrent

Haskell is concurrent. You can write an asynchronous function easily with Haskell. And Haskell has atomic threading.

Haskell has Hackage and Stackage

Hackage(with cabal) and stackage(with stack) are the most famous package servers of Hackage. They are useful and open source.
